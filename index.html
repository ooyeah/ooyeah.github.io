Fabric 找不到dSYMs文件

发表于 2016-09-01

项目加入Fabric后发现有崩溃，然后去Fabric查看的时候发现

It looks like we are missing dSYMs to process crashes for the versions below.

所以定位不了错误

原因是 Debug没有生成dSYM文件

###解决方案
1.Build Settings 搜索dsym
2.修改Debug的值为DWARF with dSYM File

Done!

git 多个ssh key如何管理（github,gitlab,oschina等）

发表于 2016-09-01

###问题解决
在~/.ssh目录下新建一个config文件配置一下，就可以解决问题

1 生成github 的ssh key

ssh-keygen -t rsa -C “yourmail@gmail.com”
在第一个对话的时候写个名字区别，例如id_rsa_github，注意看你当前路径是不是在.ssh/下，否则路径请填绝对路径
然后可以两个回车

记得添加私钥
ssh-add ~/.ssh/id_rsa_github

2 生成gitlab 的ssh key

同1，起个gitlab的名字区分开
然后两个回车

记得添加私钥
ssh-add ~/.ssh/id_rsa_gitlab

3 在.ssh/下新建一个文本，名字为config

4 在github和gitlab上添加公钥

这里不用多说

5 测试

ssh -T git@github.com

成功的话会输出
